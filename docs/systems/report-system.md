# 報告システム

## 概要

プレイヤーがバグを報告すると、ゲーム内で**詳細なバグチケット**が自動生成される。

---

## バグチケット項目

| 項目 | 説明 |
|------|------|
| 発生条件 | どんな状態のときに起きたか |
| 再現手順 | 手順を順番に記載 |
| 期待結果 | 本来どう動くべきか |
| 実際結果 | 実際に何が起きたか |
| 重要度 | クリティカル / 高 / 中 / 低 |
| 環境情報 | バージョン・プラットフォーム等 |

---

## 学習効果

プレイヤーは自然に「こういう報告を書くと伝わりやすい」という感覚を習得できる。

ただし**教材感は前面に出さない**。チケットはあくまでゲーム内UIとして機能させる。

---

## 報告品質とスコアの関係

- 必要項目が揃っているほど報告精度スコアが上がる
- 再現手順が具体的なほど再現性スコアが上がる
- 重要度の判断が正確なほど重大度評価が上がる

→ 詳細は [ランクシステム](../game-design/rank-system.md) を参照
