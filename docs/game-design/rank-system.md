# ランクシステム・評価項目

## デバッグランク

報告品質によってランクが上昇する。

```
C → B → A → S
```

ランクはエンディング内容・エンジニアの台詞に影響する。

---

## 評価項目

| 項目 | 説明 |
|------|------|
| 発見数 | どれだけのバグを見つけたか |
| 重大度 | 発見したバグの深刻さ |
| 再現性 | 報告したバグが実際に再現できるか |
| 報告精度 | 必要な情報が揃っているか |
| 情報量 | 再現手順や環境情報の網羅度 |

---

## 設計方針

単に「バグを多く見つけること」だけを評価しない。

**伝わる報告をすること**が高ランクへの条件となるよう設計する。
曖昧な報告は再現性・報告精度で減点され、
丁寧に整理された報告が総合スコアを引き上げる。
