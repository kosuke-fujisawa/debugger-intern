# エンディング

## 概要

インターン最終日の表彰シーン。

プレイヤーのランク（C / B / A / S）に応じて評価内容・台詞が変化する。

---

## ラストシーン

最後は、ずっと報告をレビューしてくれた開発エンジニアから言葉をかけられる。

> 「バグを見つけるだけじゃない。"ちゃんと伝える"って、かなり難しいんだ。」
>
> 「でも君は、それをちゃんとやってた。」

---

## 設計方針

エンディングはゲーム全体のテーマを回収する場として機能させる。

「観察して伝える」ことの価値をエンジニアの台詞を通じて言語化し、
プレイヤーが自分の体験を振り返れる余韻を持たせる。

ランクによって台詞のトーンや評価の温度感を変えるが、
いずれのルートでも後味は前向きになるよう設計する。
