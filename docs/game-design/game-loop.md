# ゲームループ・ゲーム性

## 基本ループ

```
1. プレイ・調査
   └─ ゲーム内ゲームを操作し、不具合を探す

2. 報告
   └─ 発見した内容をバグ報告として提出する

3. フィードバック
   └─ 開発エンジニアから質問やレビューが返る

4. 評価
   └─ 報告品質・重大度・再現性などからスコア化される
```

---

## ゲーム性の核：「観察」

重要なのは以下を記録すること：

- どのタイミングで起きたか
- 何をしていたか
- 毎回起きるか
- 特定条件か

---

## 推理・脱出ゲーム的構造

単純なクリック探索ではなく、

- 「なぜ起きたのか」
- 「どこまで条件を切り分けられるか」

を考える構造を持つ。

そのため、**軽い推理ゲーム・脱出ゲームに近い感触**を持つ。
